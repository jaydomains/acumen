## What was decided in this PR

- **Spec-clarification posture: combined, no F0 phase.** Per user
  direction during plan-mode, this PR codifies the frontend stack
  itself as **AC-CD19** in `CODE_SPEC.md` — exercising the
  SESSION_START.md "AC-CD-level structural additions may fold into
  the phase handover" carve-out for an addition that is materially
  larger than the PR-004 / PR-008 precedents but is well-rationalised
  against AC-CD1 (pin discipline), AC-CD2 (layout discipline), AC-CD5
  (auth-seam discipline), and AC-CD6 ("the frontend spec consumes
  this contract"). ROADMAP.md and CHECKLIST.md are intentionally
  untouched — the question of whether frontend work uses ROADMAP
  phases is deferred until follow-up PR cadence makes the answer
  obvious. PR titles use slug names (PR-032 frontend-scaffold), not
  phase identifiers.
- **AC-CD19 — frontend stack lock.** Next.js 15 (App Router), Node
  22 LTS, pnpm (packageManager-pinned), TypeScript with
  `strict: true` + `noUncheckedIndexedAccess` +
  `noImplicitOverride` + `exactOptionalPropertyTypes`, Tailwind v4,
  shadcn/ui (Radix-primitive source-copy install — reserved folder
  only in this PR), react-hook-form + zod, TanStack Query v5,
  **`openapi-typescript` (types-only) + `openapi-fetch` (typed
  runtime client)** wrapped with the project's token-attach +
  401-refresh-and-retry concerns and an `unwrap()` helper over the
  AC-CD6 error envelope, Vitest, ESLint + Prettier. Every dep pinned
  exact per AC-CD1's spirit. AC-CD19's body is self-contained: a
  fresh contributor reading only AC-CD19 should know the full stack
  and where it lives.
- **Auth storage: memory + localStorage split.** Access token in JS
  memory (cleared on reload — narrows XSS exfil window); refresh
  token in `localStorage` under `acumen.refresh_token` (persists
  across reloads). Backend stays unchanged — tokens still flow in
  the JSON body, `Authorization: Bearer …` on the header. On 401,
  a single de-duped refresh attempt fires; on refresh failure,
  tokens are cleared and the auth context flips to
  `unauthenticated`. AC-CD19 body documents the **v1.x upgrade
  path** to httpOnly Secure SameSite=strict cookies (coordinated
  backend + frontend + CORS-credentials + CSRF work) for the day the
  threat model warrants it.
- **CORS: `allow_credentials=False`.** Tokens travel in the
  Authorization header, so credentialed CORS is not needed. The
  v1.x cookie upgrade flips this. Allow-list is env-driven
  (`CORS_ALLOWED_ORIGINS`, comma-separated) so production
  deployments set the real origin explicitly.
- **OpenAPI codegen: types-only + committed snapshot.** Generated
  `src/types/api.d.ts` is committed; the source snapshot at
  `openapi/schema.json` is committed; CI regenerates against the
  snapshot and asserts zero drift via `pnpm codegen:check`. This
  keeps the frontend CI build offline (mirrors AC-CD15's no-network
  spirit). `pnpm codegen:live` is the opt-in path for regenerating
  against a running backend when the API surface changes.
- **Tailwind pinned at 4.3.0**, not the initially-planned 4.0.x.
  The 4.0.x line ships a `@tailwindcss/postcss` with an older
  oxide-scanner protocol than what current pnpm-resolved
  `@tailwindcss/oxide` expects; pinning all three (`tailwindcss`,
  `@tailwindcss/postcss`, transitively `@tailwindcss/oxide`) at
  4.3.0 resolves the "Missing field `negated` on
  ScannerOptions.sources" build failure. Caught during scaffold
  verification; called out here so a future bump audits these three
  packages together.
- **Placeholder home page is the wiring proof, not a real page.**
  `src/app/page.tsx` calls `/healthz` (proves CORS for an
  unauthenticated GET) and `/v1/auth/me` via the auth context
  (proves typed client + refresh flow + CORS for an authenticated
  GET). The "paste a refresh token in DevTools localStorage" branch
  is debug-only — it disappears when PR-033 lands the login UI.
- **structure_gate.py needs no edit.** The gate enforces a whitelist
  of required backend paths; extra top-level directories pass
  through unchallenged. `frontend/` adds no missing-path findings
  and the `app/main.py` import check doesn't trip on
  `fastapi.middleware.cors.CORSMiddleware` (not in the forbidden-
  prefix list). Surfacing here explicitly so future structure-gate
  changes know AC-CD2/AC-CD17 territory was deliberately untouched.
- New anchors introduced by this PR: **AC-CD19**.
- Existing anchors this PR depends on: **AC-CD1** (every dep
  pinned — mirrored on the frontend side), **AC-CD2** (SiteMesh
  layout — `frontend/` is the peer-of-`app/` location), **AC-CD5**
  (auth seam — the v1.x cookie upgrade pairs with this seam),
  **AC-CD6** (`/v1` REST + uniform error envelope + the typed
  client consumes the OpenAPI surface), **AC-CD15** (no-network-in-
  tests spirit — the snapshot-driven codegen mirrors this on the
  frontend), **AC-CD17** (HANDOVER_TEMPLATE.md verbatim — this
  handover is the precedent for frontend PRs and uses the template
  unchanged).

## Drift flags raised and how they were resolved

- **Stack-lock-not-in-CODE_SPEC was a "stop and surface" event.**
  SESSION_START.md lines 159–167 list "a library or pattern choice
  not already in CODE_SPEC.md" under "What is never silently
  resolved" — those normally become a user-authored spec-
  clarification PR with a fresh session implementing afterward.
  **Surfaced during plan-mode, resolved by user direction**: this
  PR is the combined spec-clarification + scaffold under the
  SESSION_START.md lines 75–85 fold-in carve-out. Documented under
  "What was decided" above.
- **ROADMAP phase naming for frontend.** Plan-mode initially
  proposed an F0 phase. **Resolved during plan-mode**: deferred —
  the phase-vs-per-PR question waits for follow-up PR cadence to
  inform the answer. ROADMAP.md and CHECKLIST.md untouched.
- **Tailwind v4.0.0 build break.** Discovered during scaffold
  verification (the `next build` step failed with a native-scanner
  protocol mismatch). **Resolved before merge**: bumped
  `tailwindcss` and `@tailwindcss/postcss` to 4.3.0. The clean
  install with the bumped versions produced a green `next build`.
  Called out under "What was decided".
- **Initial hand-rolled wrapper returned `Promise<unknown>` —
  surfaced by Gitar review on PR-032, resolved before merge.**
  The first cut of `frontend/src/lib/api/client.ts` typed the path
  parameter against `paths` but returned `Promise<unknown>`,
  forcing `as UserResponse` casts at every call site (one such
  cast in `auth/context.tsx`). Gitar's review framed it as a
  scaffold trade-off worth a follow-up, but per user direction the
  scaffold being precedent-setting means the typed-client seam has
  to deliver type safety from the start — deferring would force
  every page-building PR to cast unsafely and then refactor.
  **Resolved by adopting `openapi-fetch`** (same author as
  `openapi-typescript`, library-grade conditional types over
  `paths`, pinned exact) and amending AC-CD19 in place to record
  the openapi-typescript + openapi-fetch pairing. The wrapper
  retains only the two cross-cutting concerns the project needs
  (token attach + 401 refresh-and-retry) plus an `unwrap()` helper
  that maps the AC-CD6 error envelope onto a typed `ApiError`.
  Callers now read `await unwrap(client.GET("/v1/auth/me"))` with
  full type inference and no casts. The home page was updated to
  match.

- Known traps:
  - **The "paste a refresh token in DevTools" affordance on
    `src/app/page.tsx` is debug-only and is superseded by the
    login UI in PR-033.** When PR-033 lands, the debug paragraph
    in the `unauthenticated` branch of the home page should be
    removed (it's not a free-standing component, just a `<p>` in
    `page.tsx` — easy delete). The localStorage key
    `acumen.refresh_token` itself stays — that's the production
    storage location, not a debug artifact.
  - **AC-CD19 body is the single source of truth for the frontend
    stack.** Adding a runtime dependency is an AC-CD19
    amendment (mirror the AC-CD1 backend gravity); trivial
    dev-only additions may fold into a PR handover per the
    SESSION_START.md structural-additions carve-out. A second
    component library — Material UI, Chakra, Mantine, anything —
    is a hard amendment, not a fold-in.
  - **`exactOptionalPropertyTypes: true` is stricter than the
    Next.js default.** Building `RequestInit` (and similar
    `?:`-shaped types from the standard lib) requires conditional
    property assignment, not `{ key: x ?? undefined }`. The
    `client.ts` doFetch helper shows the pattern; copy it when
    adding similar wrappers.
  - **Tailwind 4 needs `tailwindcss`, `@tailwindcss/postcss`, and
    the transitively-resolved `@tailwindcss/oxide` to agree on
    minor version.** A pin bump that touches one without the
    other can reintroduce the "Missing field `negated` on
    ScannerOptions.sources" failure. Verify with `pnpm build`
    after any Tailwind bump.
  - **The OpenAPI snapshot is the offline source of truth.**
    Running `pnpm codegen` (default) regenerates from
    `openapi/schema.json`, NOT from a live backend. CI fails the
    build if a regeneration of the snapshot doesn't equal the
    committed `src/types/api.d.ts`. When the backend API surface
    changes, the same PR must update both the snapshot and the
    generated types (`pnpm codegen:live` followed by refreshing
    the snapshot from the running backend's `/openapi.json`).
  - **`structure_gate.py` was deliberately not edited** — it's
    permissive enough for `frontend/` to coexist. Resist the urge
    to whitelist `frontend/` there; if a future structural change
    needs it, that's AC-CD2/AC-CD17 territory and warrants its
    own surface-and-decide pass.
