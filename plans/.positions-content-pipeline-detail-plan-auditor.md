### A. Faithfulness to ratified DP rulings

- **POS-1 — Ratified DPs reflected as ratified, not re-surfaced.** Every settled DP (DP-1…DP-11
  per the resolutions above) must read in the detail plan as a **ratified** decision the slices
  build against — NOT re-surfaced as "awaiting spec-author ruling." Re-surfacing a settled DP is
  a faithfulness defect (re-litigation); equally, **baking** a posture the spec author did *not*
  rule is a surface-not-bake violation. *(yardstick: §7 + ratified resolutions)*

- **POS-2 — DP-4 gated-pilot posture is actually WIRED; no slice assumes live autonomy. [CRITICAL axis]**
  DP-4 ratified = **autopublish DISABLED, `/approve` queue** as the pilot publish path. A detail
  slice must concretely wire this: P1.1 **wires** the drain (loop closeable) but autonomous
  publish is **NOT enabled** for the KBC pilot — content reaches `create_pill` only via the
  `/approve` human gate. **No slice may assume cron-driven live autonomous publish in the pilot.**
  The E2E test (P1.5) must exercise the **gated** path, not an autonomous-publish path. The
  wire-vs-enable invariant (parent §4.1) is now resolved to *wired, gated-not-enabled for pilot*.
  *(yardstick: DP-4 ratified; parent §4.1 wire-vs-enable; audit P1-#3 / Pass1 Axis 10)*

- **POS-3 — DP-9 (cost-idempotency P1-#7) stays OUT of execution scope.** Ratified deferred.
  If the detail plan folds a cost-idempotency key into P1.1 (or any slice) as in-scope execution
  work, that is scope creep against the ratified defer. P1.1 may carry timeout/retry (P1-#8,
  folded) but **not** the idempotency key. *(yardstick: DP-9 deferred; parent §9 note, §10)*

- **POS-4 — DP-10 hybrid-paywall posture reflected in P3.1.** P3.1 content-validity gate must
  reflect the ratified **hybrid paywall** posture (flag-and-admit hybrid), not a pure-reject

  planner should write detail against the ruling and surface any *residual* mechanism question —

- **POS-5 — v1.10 versioning carried through P5.1.** P5.1 SESSION_START refresh targets **v1.10**
