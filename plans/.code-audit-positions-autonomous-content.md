**Findings (open):**
- **CA-A2-1 — Push-back (medium, adjudicable) · `corpus_builder.py` corroboration + `CorpusChunk.corroboration_count`.**
  *Uniquely content-lane — neither Gitar nor the overseer flagged it.* DS2-b(ii) corroboration counts
  distinct sources sharing an **exact `content_hash`** (byte-identical ~500-token chunk). Independent
  authoritative sources almost never emit byte-identical chunks, so `corroboration_count ≥ 2` rarely
  fires (only on verbatim-republished standard text) — the ratified AC-CD25 intent *"materially
  stronger grounding for safety content"* is largely unrealized. The test only reaches count=2 by
  feeding **identical** HTML to both sources. AC-CD25 says *"the same claim/fact"*, not "byte-identical
  chunk." Ask the executor: confirm exact-hash as the intended v1 floor (then document the limitation;
  the *"same claim/fact"* granularity is arguably a spec-author clarification, parallel to NS-3) **or**
  use embedding cosine-similarity (the embeddings already exist). Adjudicable — concede if exact-hash is
  the confirmed v1 floor. Round-trip 1/5.
- **CA-A2-2 — Push-back (BLOCKING) · `_fetch_body` `corpus_builder.py:~76` + `_gather_candidates`.**
  `httpx.AsyncClient(..., follow_redirects=True)` with **no re-validation of the final (post-redirect)
  host** against the allowlist. An allowlisted source that 3xx-redirects to a non-allowlisted/internal
  host (e.g. `169.254.169.254`) is fetched, extracted, embedded, and persisted as a `CorpusChunk`
  stamped with the **original allowlisted** `source_host`/`authority_tier`. **Violates AC-D28** (*"a host
  not on the allowlist … is not an eligible corpus source"*; ruling 3 *"web search restricted to the
  allowlist"*) and is an **SSRF / corpus-poisoning / authority-laundering** vector. Independently
  verified against the code; **≡ Gitar #1 / overseer OV-A2-17.** Fix: re-validate `response.url.host`
  via A1 `is_allowlisted` post-redirect (drop if escaped), or `follow_redirects=False` + skip 3xx;
  persist the **final** host's re-resolved tier. Round-trip 1/5.
- **CA-A2-3 — Refine (medium) · `_fetch_body:~82`.** `body = response.content[:_HTTP_FETCH_MAX_BYTES]`
  truncates **after** httpx has buffered the entire body, so the 5 MB cap does **not** bound download
  memory — the *"bound a malicious/huge payload"* comment overstates the protection, and a hostile
  allowlisted/redirect target serving multi-GB OOMs the worker. Couples with CA-A2-2. Fix: stream
  (`client.stream`) and stop at the cap. **≡ Gitar #2.** Round-trip 1/5.
- **CA-A2-4 — Refine (low, NON-blocking) · `_already_stored` per-chunk.** N+1 `SELECT`s (one per
  candidate chunk); batch the `(source_host, content_hash)` existence check into one `IN`/tuple query.
  Efficiency only. **≡ Gitar #3.** Round-trip 1/5.
