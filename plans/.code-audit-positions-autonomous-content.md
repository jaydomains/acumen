### Category 1 — faithful encoding of A1's spec contract

- **P-A1-1** — Module `app/domain/source_authority.py` exists, a **pure, offline** domain primitive:
  no DB table, no migration, no network, no provider call, no I/O. (AC-D28 Impl: *"pure, offline
  source-authority registry … carries no DB table at A1"*; detail §1.2a/§1.6.)
- **P-A1-2** — A `Tier` type enumerating T1/T2/T3 with intrinsic ordering **T1 > T2 > T3**. (Detail
  §1.2a leans `IntEnum T1=3/T2=2/T3=1`; the *ordinal values* are a detail-plan build choice, not a
  ratified value — only three-tier identity + monotone ordering is contract-binding.)
- **P-A1-3** — `authority_tier(url_or_host) -> Tier | None`: normalises to host, **lowercases**,
  **strips a leading `www.`**, matches `_ALLOWLIST` by **exact host first, then suffix-wildcard**
  for `*.<suffix>`; `*.gov.za` matches `dol.gov.za` **and** the apex `gov.za`; **exact wins over
  wildcard on tie**; non-allowlisted host → **`None`**. (AC-D28 Decision ¶ "Tier resolution".)
- **P-A1-4** — Seed allowlist matches AC-D28 **verbatim** (these are RATIFIED, not leans):
  T1 = {`iso.org`, `nrcs.org.za`, `sabs.co.za`, `*.gov.za`};
  T2 = {`nace.org`, `osha.gov`, `iec.ch`, `astm.org`};
  T3 = minimal seed (env-widened). (AC-D28 Decision bullets, DS1-b ratified.)
- **P-A1-5** — `authority_score(tier) -> float` returns **T1=1.0 / T2=0.6 / T3=0.3** exactly
  (DS1-a ratified into AC-D28), monotone. (AC-D28: *"tier→score map (T1=1.0 / T2=0.6 / T3=0.3)"*.)
- **P-A1-6** — `is_allowlisted(url_or_host) -> bool` ≡ `authority_tier(...) is not None`.
  (Detail §1.2a; AC-D28 names `is_allowlisted`.)
- **P-A1-7** — `filter_to_allowlist(results: Iterable[WebSearchResult]) -> list[tuple[..., Tier]]`:
  drops every non-allowlisted row, pairs each survivor with its `Tier`, **preserves ordering**. A1
  ships the **primitive only**; **A2** wires it around `get_web_search_source().search(...)`.
  (AC-D28: *"filters an arbitrary web-search result list to allowlisted hosts"*; detail §1.2a/§1.6.)
- **P-A1-8** — Env-extension in `app/config.py`: `source_authority_t1_extra`/`_t2_extra`/`_t3_extra`

  silently re-tier** a seed host; a host appearing at two tiers resolves to the **stronger** tier
