### S1-P1 — Input contract: topic→N, **not** name+desc→1 (the generator/refiner distinction)
A correct Slice 1 must specify the generator's **input** as a *topic prompt* (+ optional parent
subject, target count, difficulty envelope, seed reference material), explicitly **distinct** from
the built **refiner's** single admin-supplied `name` + `description`. The refiner is what exists
today: `enqueue_pill_proposal(... name, description ...)` (`app/domain/catalogue.py:488-516`),
`pill_proposal.py:27-32` ("Proposed name / Proposed description"). A Slice 1 that re-uses the
refiner's name+desc input — or fails to state the new topic→N contract — recreates the §2.2
"refiner, not a generator" gap. **Measured against:** `…workstream.md:135-145`,
`catalogue.py:488`, `pill_proposal.py:22-53`.

rationale, **evidence count**, and an **explicit cited gap signal** ("must cite the specific gap
signal"; admin evaluates in 30 s). Dropping `evidence count` or the cited gap signal re-opens the
already-folded auditor finding **A-1**. **Measured against:** `SPEC.md:346,348`,
`DECISIONS.md:529,535` (AC-D21), `…workstream.md:138-145`, `…workstream.md:378` (A-1 fold).

(which returns `[]` for a topic → silently un-grounded drafts). **Measured against:** `drive_rag.py:650-690`,
