The parent plan must keep **Gate-1** (this PR's own merge) distinct from **Gate-2** (downstream amendment/execution PRs, authored & ratified separately). Expect a `plans/**`-only PR (plan doc + wake-logs) that *records* the ratified direction and *plans* the fix work, baking **no** `SPEC` / `DECISIONS` / `CODE_SPEC` / `.claude` change. Any spec/anchor body change is spec-author-authored downstream; a fresh session implements.
*Measured against:* PR #107 §"Two gates"; SESSION_START "spec drift is surfaced, never silently resolved"; role §8.3.
