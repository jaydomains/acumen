## A. §6.5 autonomous gap-detection — the core requirement

**P-1 — Three named input signals (Missing-risk).** SPEC §6.5 (SPEC.md:344) names *exactly three*
input signals: (a) recent generated questions + their pill tags; (b) recent Testee discovery
searches that **returned no good match**; (c) recent assignments where admin **manually clarified
scope**. A correct plan must consume all three, OR explicitly surface (as spec-drift) any signal
not currently persisted. It must **not** silently narrow §6.5 to a single admin-seeded candidate.
*Expected gap:* the current `app/ai/prompts/pill_proposal.py` is a **single-candidate evaluator**
(takes one proposed name+description), not a signal-mining gap-detector — so this is likely
**Missing/Partial**, and a plan that treats §6.5 as already-built would be wrong.

**P-2 — Evidence-backed output contract (Missing-risk).** SPEC §6.5 outputs (SPEC.md:346) require:
name, parent subject, suggested difficulty range, **evidence count**, brief description,
self-applied safety classification; quality bar (SPEC.md:348) = "must be evidence-backed (must
cite the specific gap signal)." The current prompt's output JSON (pill_proposal.py:9-16) has **no
evidence_count** and cites no gap signal. The plan must add an evidence count + the specific
signal citation to the output.

**P-3 — Autonomous trigger reconciled against the locked cron set (Push-back-risk).** AC-D7
(DECISIONS.md:201): "AI extends **autonomously** as new gaps surface"; §6.5 is *autonomous*
gap-detection. The plan must define how the op runs autonomously (a scheduled task), not only an
admin-on-demand call. **AC-CD7 / SPEC §8.9 lock "seven crons."** Adding an 8th cron, or changing
the beat schedule, is an AC-CD / spec change = ratification class — it must be **surfaced**, not
silently added. The plan must either fold gap-detection into an existing cron pass or surface the
addition. Either choice must be explicit.

**P-4 — Proposals queue for admin approval; never auto-create pills (Push-back-risk).** AC-D7
Implications (DECISIONS.md:205): "New pill proposals surface in admin's **review queue**"; AC-CD7
`processing_tasks` is the queue substrate. The plan must route proposals to admin approve/reject
(`processing_tasks`), never auto-insert catalogue pills. On approval, "an **incremental bootstrap
auto-runs** per AC-D23" — the plan must preserve the approve→incremental-bootstrap chain.

**P-5 — Safety self-classification preserved, both AC-D21 signals (Refine-risk).** AC-D21
(DECISIONS.md:529): proposal output schema carries `safety_relevant` self-classification (signal
b), AND keyword auto-tagging runs at pill creation (signal a). The plan must keep the proposal's
self-classification (already present at pill_proposal.py:14) AND ensure the approval/creation path
still runs keyword auto-tagging. Neither signal may be dropped.

**P-15 — "No good match" discovery signal persistence (Missing-risk).** §6.5 input (b) requires
"recent Testee discovery searches that returned no good match." If this signal is **not currently
persisted** (per §4.6 discovery / AC-D8), mining it requires new persistence (a column/table +
migration). The plan must either locate where it is persisted or surface that it must be added —
and must **not** assume the signal already exists. A new entity/column is an AC-CD structural
addition (surface it).

**P-16 — Proposal dedup / idempotency across repeated autonomous runs (Refine, derived).** An
autonomous detector that runs on a schedule must not flood the admin queue with duplicate
proposals for the same gap on each pass (defeats the "evaluate in 30 seconds" quality bar,
SPEC.md:348). The plan should address dedup against already-pending / already-rejected proposals.
*Derived from the spec's quality bar + autonomous-repetition reality, not a verbatim clause —
flagged as such.*

## B. AI-generation infrastructure invariants (apply to any AI content op the workstream adds)

**P-6 — All model calls through the `AIProvider` protocol (Push-back-risk).** AC-CD8
(CODE_SPEC.md:637) + SESSION_START "CODE_SPEC decisions never to silently violate": every model
call goes through the `AIProvider` protocol (`generate/grade/review/embed`) with resolution order
Test-override → system override → coded default. The plan must **never** call the Anthropic/OpenAI
SDK directly from a router or domain module.

**P-7 — Provider assignment held (Confirm-target).** §6 (SPEC.md:296) / §6.5: pill proposal is an
**Anthropic-primary** operation. The plan must keep pill proposal on Anthropic; it must not
silently move it cross-family (cross-family is reserved for grade review §6.6 / anchor self-review
§6.7 per AC-D19).

**P-8 — Per-call cost capture (Refine-risk).** §6 Cost tracking (SPEC.md:374-376): every AI call
records provider, model, input/output tokens, cost estimate, and aggregates (here to system /
bootstrap / a pill-proposal bucket). The plan must capture per-call cost on every new AI op it adds.

**P-9 — Silent retry, no Testee-facing failure (Confirm-target).** §6 Error handling
(SPEC.md:385): "For weakness, learning material, **pill proposal**, anchor self-review: silently
retried; no Testee-facing impact." The plan must use retry-with-backoff (`tenacity`, per stack)
and surface no Testee-facing failure for pill proposal.

**P-10 — Prompt is a VCS registry; version bumped + persisted (Push-back-risk).** AC-CD8: prompts
live in `app/ai/prompts/` with an embedded `VERSION`; the version used is persisted **on the
resulting entity row**, never global. Since the plan must change the pill-proposal prompt (to
consume gap signals), it must **bump `VERSION`** and persist it on the proposal task row. A prompt
change with a static VERSION is a finding.

**P-12 — Domain-layer placement + coverage, no network in tests (Refine-risk).** AC-CD2 layering
(thin routers → `app/domain/*` → models, CODE_SPEC.md) + AC-CD15: `app/domain/*` carries near-full
unit+branch coverage with fixtures derived from DECISIONS formulas; `conftest.py` forbids network
calls in tests. Gap-detection logic must live in `app/domain` (e.g. `catalogue.py` or a new module
folded per the structural-additions carve-out), with unit coverage and AI calls mocked.
